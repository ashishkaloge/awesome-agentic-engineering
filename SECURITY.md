# Security Policy

This repository contains Markdown documentation and GitHub configuration. It
does not contain runtime code, production infrastructure, private project files,
or secrets.

## Supported Versions

The default branch and latest release are supported.

## Report a Security Issue

Please do not open a public issue for security-sensitive concerns such as:

- A link that appears to host malware, credential theft, or license bypasses.
- A resource that promotes abusive automation, spam, scraping abuse, or bulk messaging.
- A submitted prompt, rule, skill, or plugin that appears to exfiltrate data or weaken sandboxing.
- Any accidental secret, private URL, or private project artifact.

Use GitHub private vulnerability reporting if it is enabled, or contact the
maintainer privately.

## Safety Expectations

- Keep the repository documentation-only.
- Prefer canonical public links.
- Remove suspicious, stale, or access-gated links quickly.
- Do not add generated dumps, binaries, archives, logs, or deployment exports.
